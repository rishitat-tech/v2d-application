EGO Guide

EGO Documentation

This documentation covers the EGO recording setup, pre-built bundle workflow, recording rules, movement principles, and backup build-from-source option.

Use Pre-built Bundle Recommended

The recommended setup is to use the pre-built EGO camera collection bundle. This bundle includes the orchestration script, requirements file, IsaacTeleop wheel, and OAK camera plugin folder.

Expected Folder Structure

egocam_collection_bundle/
├── egocam_collection.py
├── requirements.txt
├── isaacteleop*.whl
└── oak_camera/
    └── ...

There are both x86_64 and arm64 bundles. Download the correct bundle, unzip it, and confirm the folder structure before running.

Pre-built Bundle Setup

At the root directory of egocam_collection_bundle/, create and activate the Python 3.11 virtual environment:

uv venv --python 3.11 venv_isaacteleop
source venv_isaacteleop/bin/activate

Install dependencies:

uv pip install -r requirements.txt

Run Recording

With Preview and OSMO Bucket Upload

uv run egocam_collection.py --bucket-path s3://recordings/v2d/ego/ --endpoint-url https://pdx.s8k.io --access-user '' --secret-key '' --region us-east-1 --preview

With Preview and No Upload

uv run egocam_collection.py --no-upload --preview

The script will prompt for a session ID. Leaving it empty uses the timestamp as the ID.

It will then prompt to select streams. OAKD-W supports Color, MonoLeft, and MonoRight. Leaving the stream selection empty will select Color.

Press Ctrl + C to stop recording.

A raw H264 frame file and MP4 mux for each selected stream will be saved under: 

ego-reconstruction/collection/recordings/

Naming convention:

SessionID_{color|monoleft|monoright}.{h264|mp4}

MP4 files are uploaded to the configured bucket path under the session ID.

Rule of Thumb When Recording

When recording, prioritize safety and smooth, deliberate movements above all else. Treat the hardware with care and do not deliberately force anything.

Start Pose and Visibility Requirements

  • At the start of each session, hold the gesture with palms facing each other and fingers bent.
  • Hold the start pose steadily for 3 seconds.
  • At the end of the session, return to the same pose.
  • Make sure wrists are visible in the camera field of view.
  • Always make sure fingers and wrists are fully visible and not covered by clothes.
  • Make sure no other person or extra hand is visible inside the head-mounted camera view.

Key Movement Principles

  • Go Slow: Jerky or quick movements often lead to errors and auto-deletions.
  • Natural Grasp: Use smooth, consistent, natural, and deliberate grasp techniques.
  • Consistent Speed: Keep a consistent speed throughout the video.
  • Smooth Continuous Movements: Movements should be fluid and task-related without unnecessary pauses.
  • Minimal Waist Rotation: Avoid unnecessary rotation that makes the video unstable.
  • Avoid Making a Fist: When not interacting with objects, keep fingers relaxed and visible.

Movement Speed and Duration

Target duration is 10–40 seconds minimum per episode unless stated otherwise.

  • Teaching Speed: Slowest speed with exaggerated clarity. Default for all collection.
  • Standard Speed: Measured and purposeful pace. Use when specified.
  • Natural Speed: Real-world practical speed. Use only when explicitly requested.

Build from Source Backup Option

Building from source is a backup option and requires source repository access.

Setup Summary

git clone https://github.com/NVIDIA/IsaacTeleop.git
cd IsaacTeleop

cmake -B build
cmake --build build
cmake --install build

export ISAAC_TELEOP_DIR=/path/to/IsaacTeleop
pip install "$ISAAC_TELEOP_DIR/install/wheels/"*.whl

pip install av
pip install boto3

The orchestration script is used from the ego reconstruction collection workflow. Make sure the OAK-D camera is connected with a USB 3.0 cable to the host machine. 

python egocam_collection --no-upload

Press Ctrl + C to stop recording.

Use the pre-built bundle whenever possible. Build from source should be used as a backup path only.
