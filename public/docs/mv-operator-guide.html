MV Operator Guide

This guide explains the operator workflow for connecting to the Nova Orin, launching the recorder, checking the device IP address, and opening the monitoring UI.

SSH into Nova Orin

On your work computer connected to the required company network, open a terminal and SSH into the Nova Orin.

Do not store plaintext passwords in this guide. Use the approved shared credential source or ask the project lead for the current Nova Orin password. 

ssh nvidia@nova-devkit-6.dyn.nvidia.com

If host verification appears, type: 

yes

When prompted, enter the approved Nova Orin password. 

Password: <Nova Orin password>

Start tmux and Enter Isaac ROS Dev Environment

Once inside the Nova devkit shell, start a tmux session and enter the Isaac workspace. 

tmux
cd /mnt/nova_ssd/workspaces/isaac
./scripts/run_dev.py

After this, you should be inside the container shell. It may take a minute. The prompt should look similar to: 

admin@nova-devkit-6:/workspaces/isaac_ros-dev/ros_ws

Source the workspace: 

source install/setup.bash

Alternative: Install Pre-built Packages from APT

Instead of building ROS packages from source, pre-built packages can be installed from APT directly. However, this requires reinstalling each time the container is launched. 

tmux
cd /mnt/nova_ssd/workspaces/isaac
./scripts/run_dev.py
sudo apt-get update
sudo apt-get install -y ros-humble-isaac-ros-nova-recorder

If using the APT package flow, do not run source install/setup.bash.

Launch Nova Recorder

All necessary Isaac ROS packages should already be built, so launch the recorder command below. You may need to try this multiple times because the current build can run into a buffer out-of-memory error. Once it successfully launches, it should continue without issues. 

ros2 launch isaac_ros_nova_recorder nova_recorder.launch.py config:=hawk-4

Get the Orin Local IP Address

Open a separate terminal inside the Nova Orin by using: 

Ctrl + b, then c

Run: 

ifconfig

Look for the local IP address under the network interface, for example: 

eno1: flags=4163<UP,BROADCAST,RUNNING,MULTICAST>  mtu 1500
        inet 10.xxx.xx.xxx  netmask 255.255.254.0  broadcast 10.xxx.xx.xxx

Open the Monitoring UI

Once the IP address has been obtained, open Chrome and visit the URL below. Replace <local_ip> with the Orin IP address from ifconfig. 

https://<local_ip>:8080/?ds=foxglove-websocket&ds.url=ws%3A%2F%2F<local_ip>%3A8765

To move back and forth between tmux terminals, use: 

Ctrl + b, then p

Successful UI Connection

A successful connection should bring you to the monitoring UI.
