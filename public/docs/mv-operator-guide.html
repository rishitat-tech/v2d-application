MV Operator Guide

This guide covers MV operator responsibilities, recording flow, communication with the data collector, and troubleshooting steps for Foxglove, Nova Orin, Docker, and camera issues.

Operator Responsibilities

  • Complete required recording fields such as Author, Title, and Location.
  • Use the correct task title based on the dataset/task being collected.
  • Start the recording before the data collector begins moving.
  • Give a 3 second countdown before signaling the data collector to start.
  • Confirm the data collector stays inside the marked bounding area.
  • Confirm the target object is visible and not blocked from camera views.
  • End the recording when the data collector indicates the sequence is complete.
  • Communicate clearly when the session has ended.

Before Recording

  • Verify the calibration video is good quality before starting data collection.
  • Confirm the correct pre-scanned object is being used.
  • Make sure only the required object is present.
  • Confirm the data collector is inside the marked bounding area.
  • Make sure the target object is not blocked from any camera view.

Foxglove Troubleshooting

Connection Issues

If there is an issue when attempting to connect to Foxglove, try reconnecting first. If the issue continues, clear the browser cache and reconnect using: 

<IP Address>:8080

Use the IP address with port 8080 instead of the full URL.

Foxglove Delay

If Foxglove appears delayed, refresh the Foxglove session first. If refreshing does not solve the issue, reboot the Nova Orin.

Rebooting Nova Orin

In the terminal where the process is running, press: 

Ctrl + C

Then run:

sudo reboot

After rebooting, SSH back into the Nova Orin, enter the tmux session, and run: 

tmux
argus_syncstereo
nova_preflight_checker

Then proceed to activate the Isaac ROS development environment again.

Python Package Fixes

If issues arise related to YAML, PyYAML, or packaging, run:

python3 -m pip uninstall -y yaml pyyaml
python3 -m pip install pyyaml
python3 -m pip install packaging

Docker / Isaac ROS Recorder Troubleshooting

If the issue appears related to Docker or the Isaac ROS recorder setup, run: 

docker system prune -a

After this, exit Docker and continue with: 

cd /mnt/nova_ssd/workspaces/isaac
./scripts/run_dev.py
sudo apt-get update
sudo apt-get install -y ros-humble-isaac-ros-nova-recorder

Do not source install/setup.bash.

argus_syncstereo Troubleshooting

If the following command is not working: 

argus_syncstereo

Check the status of the service:

sudo systemctl status nvargus-daemon

If the system is inactive, restart it:

sudo systemctl restart nvargus-daemon

After restarting, proceed with the normal commands again.

Quick Troubleshooting Checklist

  • Refresh Foxglove if it appears delayed.
  • Clear browser cache and reconnect using <IP Address>:8080.
  • Reboot Nova Orin if refresh does not resolve the issue.
  • Restart tmux workflow after SSH reconnect.
  • Run argus_syncstereo and nova_preflight_checker.
  • Fix YAML/PyYAML package issues if needed.
  • Use Docker prune and reinstall recorder package if Docker/recorder issues occur.
  • Check and restart nvargus-daemon if argus_syncstereo fails.
